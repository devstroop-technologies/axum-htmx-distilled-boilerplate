About This Boilerplate

Architecture

A full-stack Rust web application with SPA-like capabilities, no JavaScript framework required.

  • Backend: Axum — async Rust web framework built on Tokio
  • Frontend: Server-rendered HTML templates with HTMX for dynamic updates
  • Templates (Dev): minijinja — hot-reloads from disk, edit and refresh
  • Templates (Release): askama — compiled into the binary, zero overhead
  • Styling: Custom CSS (~5KB) with CSS custom properties design system
  • Reactivity: Vanilla JS — zero framework dependencies
  • API: JSON REST endpoints with OpenAPI (Swagger UI)
How HTMX SPA Works
  1. Browser navigates to /, /about, etc. — server returns a full HTML page
  2. HTMX attributes like hx-get="/partials/status-card" fire requests that return HTML fragments
  3. HTMX swaps the fragment into the page — no full reload, feels like a SPA
  4. Polling (hx-trigger="every 10s") keeps data live without WebSockets
Project Structure
src/
  bin/main.rs          ← Entry point, router, middleware
  lib.rs               ← Library root with macro exports
  config.rs            ← TOML config loader
  error.rs             ← Typed errors with HTMX-aware responses
  render.rs            ← define_page! / define_partial! macros
  handlers/
    templates.rs       ← Full-page handlers (3 lines each!)
    partials.rs        ← HTMX partial handlers
    api/health.rs      ← JSON REST API handlers
  services/
    mod.rs             ← Service container
    health.rs          ← Health service (trait + impl)
    items.rs           ← Item service (in-memory, DB-ready)
  middleware/mod.rs    ← Security headers, request logging
  models/mod.rs        ← Shared state with services
  utils/
    logging.rs         ← Tracing setup
    templates.rs       ← minijinja hot-reload helper
templates/
  base.html            ← Layout with sidebar, theme toggle
  pages/*.html         ← Full page templates
  partials/*.html      ← HTML fragment templates
  components/*.html    ← Design tokens
static/css/app.css     ← Minimal CSS framework (~5KB)
config/app.toml        ← Application configuration
